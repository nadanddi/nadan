# Investor Flow Tracker App Specification

## 1. 앱 목표

이 앱은 네이버 금융 또는 보조 데이터 소스를 기반으로 한국 주식시장에서 기관과 외국인의 매수/매도 흐름을 빠르게 확인하기 위한 도구이다.

핵심 목표는 다음과 같다.

- 기관 매수/매도 상위 종목 Top 5 확인
- 외국인 매수/매도 상위 종목 Top 5 확인
- 기관 4일 이상 연속 매수/매도 종목 확인
- 외국인 4일 이상 연속 매수/매도 종목 확인
- 연속 매수/매도 종목 목록과 누적 순매수/순매도 금액 Top 5를 분리 표시
- LG Gram 노트북 환경과 iPhone 환경에서 모두 사용 가능하게 구현

---

## 2. 핵심 사용자 시나리오

### 2.1 당일 수급 확인

사용자는 앱을 열고 오늘 기관/외국인이 많이 산 종목과 많이 판 종목을 각각 Top 5로 확인한다.

필요 화면:

- 기관 매수 Top 5
- 기관 매도 Top 5
- 외국인 매수 Top 5
- 외국인 매도 Top 5

---

### 2.2 연속 매수/매도 확인

사용자는 최근 4거래일 이상 연속으로 기관 또는 외국인이 매수/매도한 종목을 확인한다.

필요 화면:

- 기관 4일 이상 연속 매수 종목
- 기관 4일 이상 연속 매도 종목
- 외국인 4일 이상 연속 매수 종목
- 외국인 4일 이상 연속 매도 종목

---

### 2.3 연속 순매수/순매도 총량 확인

사용자는 단순히 연속 여부만 보는 것이 아니라, 연속 기간 동안 누적 순매수/순매도 규모가 큰 종목 Top 5를 확인한다.

필요 화면:

- 기관 연속 순매수 누적금액 Top 5
- 기관 연속 순매도 누적금액 Top 5
- 외국인 연속 순매수 누적금액 Top 5
- 외국인 연속 순매도 누적금액 Top 5

---

## 3. 화면 구성

## 3.1 대시보드 홈

권장 구성:

```text
[날짜 선택] [시장 선택: KOSPI / KOSDAQ / 전체]
[새로고침]

기관
- 매수 Top 5
- 매도 Top 5

외국인
- 매수 Top 5
- 매도 Top 5

연속 수급
- 기관 연속 매수
- 기관 연속 매도
- 외국인 연속 매수
- 외국인 연속 매도
```

---

## 3.2 Top 5 카드

각 카드는 다음 정보를 표시한다.

| 항목 | 설명 |
|---|---|
| 순위 | 1~5위 |
| 종목명 | 종목 이름 |
| 종목코드 | 6자리 코드 |
| 매수/매도 금액 | 당일 기준 금액 |
| 순매수/순매도 | 가능하면 순매수 기준 표시 |
| 등락률 | 선택 항목 |

---

## 3.3 연속 매수/매도 카드

단순 연속 여부 카드:

| 항목 | 설명 |
|---|---|
| 종목명 | 종목 이름 |
| 연속일수 | 4일 이상 |
| 방향 | 매수 또는 매도 |
| 최근 날짜 | 마지막 확인 날짜 |

---

## 3.4 연속 누적금액 Top 5 카드

누적 총량 카드:

| 항목 | 설명 |
|---|---|
| 순위 | 1~5위 |
| 종목명 | 종목 이름 |
| 연속일수 | 연속 매수/매도 일수 |
| 누적 순매수/순매도 금액 | 연속 기간 합계 |
| 평균 일별 금액 | 선택 항목 |

## 4. 데이터 정의

## 4.1 기본 데이터 컬럼

```text
trade_date
market
stock_code
stock_name
investor_type
buy_amount
sell_amount
net_amount
buy_volume
sell_volume
net_volume
```

## 4.2 investor_type

최소 지원 대상:

```text
institution
foreign
```

향후 확장 가능 대상:

```text
pension_fund
investment_trust
insurance
private_fund
financial_investment
individual
```

---

## 5. 연속 매수/매도 계산 규칙

## 5.1 연속 매수 조건

```text
net_amount > 0인 거래일이 4거래일 이상 연속
```

## 5.2 연속 매도 조건

```text
net_amount < 0인 거래일이 4거래일 이상 연속
```

## 5.3 단순 연속 종목 목록

정렬 기준:

1. 연속일수 내림차순
2. 최근 날짜 기준 최신순
3. 종목명 오름차순

## 5.4 연속 누적금액 Top 5

정렬 기준:

- 연속 매수: 연속 기간 누적 `net_amount` 내림차순
- 연속 매도: 연속 기간 누적 `net_amount` 오름차순 또는 절댓값 내림차순

권장 표시:

- 매수는 양수 금액으로 표시
- 매도는 절댓값 기준으로 표시하되 UI에는 `순매도`라고 명확히 표시

## 6. 데이터 수집 전략

## 6.1 1순위: 네이버 금융 기반 수집

목적:

- 사용자가 익숙한 네이버 금융의 투자자별 매매동향 정보를 기반으로 빠르게 확인한다.

주의:

- 네이버 금융 페이지 구조가 바뀌면 크롤러가 깨질 수 있다.
- 요청 빈도가 높으면 차단될 수 있다.
- 개인 사용 목적의 낮은 빈도 수집으로 시작한다.

---

## 6.2 2순위: KRX 공식 데이터 보조 사용

KRX 정보데이터시스템은 투자자별 거래실적, 투자자별 거래실적 개별종목, 투자자별 순매수상위종목 메뉴를 제공한다.

사용 목적:

- 네이버 금융 데이터 수집 실패 시 보조 데이터 소스로 활용
- 장 마감 후 확정 데이터 검증
- 향후 더 안정적인 데이터 파이프라인으로 전환

---

## 7. 권장 기술 스택

## 7.1 MVP 1단계

가장 빠른 구현:

| 영역 | 기술 |
|---|---|
| Backend/Data | Python |
| Data processing | pandas |
| UI | Streamlit |
| Storage | CSV 또는 SQLite |
| 실행 환경 | LG Gram 노트북 브라우저, iPhone Safari |

장점:

- 구현이 빠르다.
- Python 기반 데이터 처리와 잘 맞는다.
- iPhone에서도 브라우저로 접속 가능하다.

단점:

- 모바일 앱처럼 설치되는 앱은 아니다.
- 배포 서버가 필요할 수 있다.

---

## 7.2 MVP 2단계

확장형 구조:

| 영역 | 기술 |
|---|---|
| Frontend | Next.js |
| Backend API | FastAPI |
| DB | SQLite → PostgreSQL |
| Scheduler | APScheduler 또는 cron |
| Deploy | Render, Railway, Fly.io, 개인 서버 |

장점:

- 반응형 웹앱 구현이 좋다.
- iPhone 홈 화면 추가가 가능하다.
- API와 데이터 수집 로직을 분리할 수 있다.

---

## 8. 권장 구현 순서

## Step 1. 데이터 구조 만들기

- 종목별 투자자 수급 데이터를 저장할 테이블 정의
- 날짜, 종목코드, 종목명, 투자자구분, 매수, 매도, 순매수 컬럼 정의

## Step 2. 샘플 데이터로 화면 먼저 만들기

- 실제 크롤링 전에 CSV 더미 데이터로 UI 구성
- Top 5 카드와 연속 매수/매도 카드 구현

## Step 3. 네이버 금융 수집기 구현

- 단일 날짜 기준 기관/외국인 Top 5 수집
- 데이터 정규화
- 실패 시 로그 출력

## Step 4. 과거 데이터 누적

- 최근 10~20거래일 데이터를 저장
- 연속 매수/매도 계산 가능하게 구성

## Step 5. 연속 수급 계산 로직 구현

- 4일 이상 연속 매수/매도 탐지
- 단순 연속 목록과 누적금액 Top 5 분리

## Step 6. 모바일 반응형 UI 적용

- iPhone 화면에서 카드가 세로로 쌓이도록 구성
- 작은 화면에서도 종목명, 금액, 연속일수가 잘 보이도록 조정

## Step 7. 자동 갱신

- 수동 새로고침 먼저 구현
- 이후 장 마감 후 자동 갱신 추가

## 11. 주의할 점

- 이 앱은 투자 판단 보조 도구이며, 매수/매도 추천 앱으로 만들지 않는다.
- 데이터 지연 가능성을 UI에 표시한다.
- 네이버 금융 수집 실패 시 사용자에게 명확히 표시한다.
- 데이터 출처와 갱신 시간을 화면에 표시한다.
- 연속 매수/매도는 거래일 기준으로 계산한다.
- 공휴일과 주말은 연속일 계산에서 제외한다.
